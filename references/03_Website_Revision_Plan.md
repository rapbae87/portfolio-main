# 웹사이트 수정 실행 계획
**작성일:** 2026년 6월 26일  
**목적:** Brand Strategy 문서 및 홈페이지 진단을 기반으로, 다음 단계 웹사이트 수정을 위한 구체적인 실행 계획 제시  
**원칙:** 이번 단계에서는 코드를 수정하지 않는다. 이 문서는 승인 후 실행을 위한 계획서다.

---

## 실행 원칙

이번 수정의 방향은 하나다. **브랜드 에이전시 포트폴리오에서 전략 실행가의 제안서로.**

화려함보다 질서, 감성보다 논리, 장식보다 정보. 방문자가 홈페이지를 보고 느껴야 하는 것은 "이 사람의 감각이 좋다"가 아니라 "이 사람에게 브랜드를 맡기면 어떤 결과가 나올지 예상된다"여야 한다.

---

## Phase 1. 디자인 시스템 재정의

### 1-1. 타이포그래피

현재 Noto Serif KR(디스플레이) + Noto Sans KR(본문) 조합은 감성적이고 문학적인 인상을 준다. 이를 더 논리적이고 구조적인 인상을 주는 조합으로 교체한다.

| 용도 | 현재 | 변경 방향 |
|------|------|----------|
| 헤딩 | Noto Serif KR (세리프, 감성적) | IBM Plex Sans KR 또는 Pretendard (산세리프, 명료함) |
| 본문 | Noto Sans KR | Pretendard 또는 동일 유지 |
| 레이블/수치 | 현재 모노 폰트 | IBM Plex Mono 또는 Roboto Mono 유지 |

### 1-2. 색상 시스템

현재 베이지/오프화이트 계열은 감성적이다. 전략 제안서 미감을 위해 다음으로 전환한다.

| 역할 | 현재 | 변경 방향 |
|------|------|----------|
| 배경 | #f7f5f2 (베이지) | #ffffff (순백) 또는 #f8f8f8 (쿨 그레이) |
| 텍스트 | #111 | #0a0a0a (더 강한 검정) |
| 강조색 | 없음 | 단일 강조색 1개 (예: #1a1a2e 네이비 또는 #c41e3a 레드) |
| 보조 텍스트 | #666, #999 | #4a4a4a, #888 (쿨 톤) |

### 1-3. 레이아웃 원칙

- 이미지 중심에서 텍스트/데이터 중심으로 전환
- 감성적 여백보다 정보 밀도 있는 구조 선호
- 표(Table)와 그리드를 적극 활용
- 섹션 구분은 배경색 변화보다 타이포그래피와 라인으로

## Phase 2. 헤로 섹션 재설계

### 현재 문제
- 배경 이미지 + 감성 카피 구조
- 방문자가 5초 안에 역할을 이해하지 못함

### 새로운 구조 제안

```
[헤로 섹션 — 전체 화면 높이]

상단 레이블: BRAND BUILDER / STRATEGIST / EXECUTOR
메인 헤딩: 브랜드를 전략에서 실행까지.
서브 텍스트: 상품 기획, D2C 구축, 콘텐츠, 퍼포먼스 마케팅, B2B 제안까지
           하나의 흐름으로 수행합니다.

[핵심 수치 블록 — 헤로 하단]
4개 브랜드 동시 운영 | 1.1M 팬덤 구축 | 370x 성장 | 150+ OEM 벤더

[CTA]
케이스 스터디 보기 →    협업 문의 →
```

### 배경 처리
- 감성적 배경 이미지 제거
- 순백 또는 매우 연한 그레이 배경
- 좌측 정렬 레이아웃 (중앙 정렬 탈피)

## Phase 3. 역할 정의 섹션 신설

현재 홈페이지에 없는 섹션이다. 방문자가 "이 사람이 어디까지 하는 사람인가"를 이해하는 데 가장 중요한 섹션이다.

### 구조 제안

```
[섹션 헤딩]
하나의 흐름으로 브랜드를 만듭니다.

[7단계 가치사슬 시각화]
브랜드 전략 → 상품 기획 → D2C 구축 → 콘텐츠 제작 → 퍼포먼스 마케팅 → 오프라인 확장 → B2B 제안

[각 단계 설명 — 표 또는 카드 구조]
| 단계 | 수행 내용 | 실제 사례 |
|------|----------|----------|
| 브랜드 전략 | 포지셔닝, 타겟 정의, 차별화 설계 | Atlantis, Feeju |
| 상품 기획 | 카테고리 구조, SKU 설계, 드롭 전략 | 드림컴스, Atlantis |
| D2C 구축 | Shopify 기반 자사몰, CRM 운영 | Wiggle Wiggle, PETHROOM |
| 콘텐츠 제작 | 숏폼, 상세페이지, 크리에이티브 800+ | 드림컴스, 1993 Studio |
| 퍼포먼스 마케팅 | Meta, Naver, Kakao, Google | EMFLOW 전체 |
| 오프라인 확장 | 팝업, 플래그십, 리테일 파트너십 | 1993 Studio |
| B2B 제안 | 라이선스 협상, 파트너십 계약 | Atlantis, Klosterfrau |
```

## Phase 4. 케이스 스터디 섹션 재설계

### 현재 문제
- 결과 데이터 없음
- 역할 설명 없음
- 글로벌 프로젝트 미노출

### 새로운 케이스 스터디 구조

각 프로젝트를 다음 4가지 항목으로 정리한다.

```
[프로젝트 카드]
브랜드명 + 카테고리 레이블
역할: [구체적 역할 나열]
실행 범위: [수행한 작업 목록]
결과: [수치 포함 성과]
```

### 우선 노출 프로젝트 (중요도 순)

| 순위 | 프로젝트 | 핵심 수치 | 강조 포인트 |
|------|---------|----------|------------|
| 1 | Wiggle Wiggle | 18개월 370배 성장, 460K 회원 | D2C + 콘텐츠 성장 모델 |
| 2 | PETHROOM | 1.1M 팬덤, 600K D2C 회원 | 팬덤 기반 브랜드 구축 |
| 3 | Atlantis Strength | 글로벌 IP 한국 독점 라이선스 | 글로벌 IP × 한국 시장 |
| 4 | 1993 Studio | 400K 팬베이스, 스포츠 × 패션 | IP 기반 문화 브랜드 |
| 5 | Klosterfrau | 독일 150년 브랜드 × 한국 시장 | 글로벌 파트너십 |
| 6 | 로보트태권브이 | 한국 문화 IP 사업화 | IP 사업화 전략 |

## Phase 5. 성과 데이터 블록 신설

현재 홈페이지에 전혀 없는 요소다. 신뢰를 만드는 가장 직접적인 방법이다.

### 구조 제안

```
[섹션 헤딩]
숫자로 증명합니다.

[데이터 그리드 — 6개 수치]
1.1M     팬덤 구축 (PETHROOM)
370x     성장 (Wiggle Wiggle, 18개월)
800+     크리에이티브 제작
150+     OEM 벤더 운영
4개      브랜드 동시 운영
3개국    글로벌 프로젝트 수행
```

## Phase 6. 글로벌 프로젝트 강조 섹션

현재 홈페이지에서 완전히 누락된 차별화 포인트다.

### 구조 제안

```
[섹션 헤딩]
글로벌 IP를 한국 시장으로 연결합니다.

— 한국 문화 IP 사업화
— 라이선스 구조 설계, 사업 모델 개발
```

## Phase 8. CTA 섹션 재설계

### 현재 문제
- "대화 시작하기"는 너무 일반적
- 어떤 사람이 연락해야 하는지 불분명

### 새로운 구조 제안

```
[섹션 헤딩]
이런 상황이라면 연락하세요.

[3가지 상황 카드]
1. 브랜드를 처음 만들려 한다
   — 전략 수립부터 런칭까지 함께합니다

2. 글로벌 IP를 한국에서 런칭하려 한다
   — 라이선스 협상부터 D2C 구축까지 수행합니다

3. 기존 브랜드를 성장시키려 한다
   — 상품 기획, 콘텐츠, 퍼포먼스 마케팅을 통합 설계합니다

[연락처]
이메일 / 카카오톡 / LinkedIn
```

## 콘텐츠 준비 체크리스트

웹사이트 수정 전에 준비해야 할 콘텐츠 목록이다.

- [ ] 각 프로젝트별 성과 수치 확정 (공개 가능 범위 결정)
- [ ] 케이스 스터디 텍스트 작성 (역할 + 실행 + 결과)
- [ ] 헤로 섹션 메인 카피 확정
- [ ] 역할 정의 섹션 텍스트 확정
- [ ] 글로벌 프로젝트 소개 텍스트 작성
- [ ] About 텍스트 작성 (이력 포함)
- [ ] CTA 섹션 텍스트 확정
- [ ] 증명사진 고해상도 파일 준비
- [ ] 랩배 캐릭터 이미지 파일 준비
- [ ] 각 프로젝트 대표 이미지 또는 시각 자료 준비

## 주의사항

**공개 가능 데이터 확인 필요.** Wiggle Wiggle, PETHROOM 등의 성과 수치는 EMFLOW COMPANY 내부 데이터다. 공개 전 공개 가능 여부를 확인해야 한다.

**클라이언트 승인 필요.** Atlantis, Klosterfrau 프로젝트는 진행 중인 협상이다. 홈페이지에 노출하기 전 상대방의 동의가 필요할 수 있다.

**영문 버전 고려.** 글로벌 파트너를 대상으로 하는 경우 영문 버전이 필요하다. 현재 자료 중 영문 제안서(Atlantis, Klosterfrau)가 있으므로 영문 홈페이지 또는 영문 섹션 추가를 검토해야 한다.
