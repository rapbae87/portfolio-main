## 1. Feeju 마케팅 기획서 (`/home/ubuntu/upload/Feeju마케팅기획서_20260521.pdf`)

### 관찰 범위
- 1~5페이지 확인
- 6~10페이지 확인

### 핵심 디자인 언어
이 문서는 **브랜드 사이트**의 언어보다는 **기업 제안서와 전략 문서**의 문법을 따릅니다. 첫 인상은 감성적 연출보다 정보 구조가 먼저 들어오며, 슬라이드마다 '지금 어떤 단계의 설명을 읽고 있는지'를 즉시 파악할 수 있도록 설계되어 있습니다.

### 정보 구조
- 표지 → 핵심 메시지 → 목차 → 챕터 전환 → 세부 분석의 순서가 매우 명확함.
- 각 장은 **숫자 기반 장 구분**(`1.`, `2.`)으로 관리되고, 장 제목이 항상 전면에 크게 제시됨.
- 세부 슬라이드는 상단 검은 헤더 바 안에 현재 장 제목을 반복 표기하여 문맥을 잃지 않게 함.
- 정보는 긴 서술문보다 **블록 단위 요약**으로 나뉘며, 좌우 2단 또는 3단 구조가 많음.

### 여백 사용 방식
- 여백은 넉넉하지만 장식적이지 않고, **콘텐츠 블록 간 분리**를 위해 쓰임.
- 중앙 정렬보다 좌측 기준선 정렬이 우세하며, 문서 전체에 보이지 않는 정렬선이 일관되게 작동함.
- 표지/챕터 전환 슬라이드는 여백을 크게 사용해 제목 자체를 메시지처럼 읽게 만듦.

### 타이포그래피
- 한글/영문 모두 **산세리프 중심**.
- 제목은 굵고 크며, 본문은 실무 문서처럼 읽기 쉬운 크기로 절제됨.
- 제목, 소제목, 본문, 캡션의 위계가 분명함.
- 강조는 색보다 **굵기, 크기, 박스 라벨, 밑줄**로 처리하는 편.

### 페이지 구성 방식
- 검은 상단 바 + 본문 흰 배경 구조가 반복됨.
- 챕터 전환 페이지는 극단적으로 단순함: 큰 숫자와 장 제목만 배치.
- 세부 내용 페이지는 다음 유형이 반복됨:
  1. 좌측 설명 / 우측 이미지·그래프
  2. 3열 비교 분석
  3. 표 기반 정리
  4. 스크린샷 + 핵심 주석
- 따라서 웹사이트에서도 카드 갤러리보다 **문서형 섹션 모듈**이 적합함.

### 이미지와 텍스트의 균형
- 이미지는 감성 연출용이 아니라 **근거 자료**로 사용됨.
- 제품 사진, 그래프, 경쟁사 화면 캡처가 본문 논리를 보조함.
- 이미지는 항상 텍스트 설명과 짝을 이루고, 단독 장식 요소로 쓰이지 않음.

### 정보 전달 방식
- 결론을 먼저 말하고 근거를 뒤에 배치하는 **제안서형 구조**.
- 불릿, 표, 태그, 강조 박스를 사용해 빠르게 읽히도록 구성.
- 독자는 '감상'보다 '판단'을 하게 됨.

### 웹 적용 시 해석 방향
- 홈 화면도 비주얼 히어로보다 **핵심 입장문 + 짧은 설명 + 명확한 섹션 인덱스** 구조가 적합.
- Works는 썸네일 갤러리가 아니라 **문제 / 관찰 / 전략 / 실행** 순의 문서형 목록으로 재구성해야 함.
- Thinking은 블로그형 카드보다 **메모/노트/제안서 섹션처럼 정리된 목록**이 어울림.
- 전체적으로 검은 헤더 바, 숫자 기반 섹션 구분, 산세리프 타이포, 넓고 절제된 여백을 핵심 시스템으로 삼을 수 있음.

## 2. 드림컴스 브랜드 디자인 기획서 (`/home/ubuntu/upload/드림컴스브랜드디자인기획_리뉴얼.pdf`)

### 관찰 범위
- 1~5페이지 확인

### 핵심 디자인 언어
드림컴스 문서는 Feeju 제안서보다 더 직접적으로 **디자인 시스템 문서**의 성격을 띱니다. 그러나 표현 방식은 여전히 감성 중심이 아니라 **체계, 기준, 적용 원칙**을 전달하는 데 초점이 있습니다. 웹으로 옮길 때 중요한 것은 마지막 시안 화면보다 앞선 페이지들에 담긴 **기준표, 역할표, 섹션별 사용 규칙**입니다.

### 정보 구조
- 표지에서 브랜드명과 문서 주제를 한 번에 명확히 제시함.
- 이후 슬라이드는 `브랜드 방향 요약 → Tone & Color → 사용 예시`로 이어지는 구조이며, 추상적 설명보다 **정의 → 기준 → 적용 예시** 순서가 분명함.
- 섹션 제목은 번호와 함께 상단 바에 배치되어 문서의 단계가 계속 추적됨.

### 타이포그래피와 계층
- 전체가 산세리프 기반이며, 한글 본문 가독성이 우선임.
- 강조는 거대한 장식 서체가 아니라 **굵기, 상단 바, 표 구조, 열 제목**으로 형성됨.
- 제목은 명확하고 짧으며, 본문은 설명문보다 정리문에 가까움.

### 여백과 레이아웃
- 페이지 대부분이 **좌측 정렬**을 기본으로 삼고, 넓은 캔버스 안에 콘텐츠를 질서 있게 놓음.
- 여백은 무드 연출보다 **표, 텍스트, 샘플 이미지가 서로 충돌하지 않게 하는 기능적 여백**으로 보임.
- 특히 표와 기준표가 넓은 공간 안에서 안정적으로 배치되어 '생각이 정리된 문서'라는 인상을 강화함.

### 페이지 구성 방식
- 단순 소개 페이지
- 기준표/컬러 시스템 표
- 톤 앤 무드 정의 표
- 최종 적용 시안
이라는 흐름이 보이며, 웹에서도 이 구조를 참고해 **개념 → 기준 → 적용 결과**의 순차적 독서 경험을 만들 수 있음.

### 웹 적용 시 해석 방향
- 사이트 전체를 '브랜드 사이트'보다 **설계 문서**처럼 구성해야 함.
- 카드보다 표, 리스트, 메타 데이터 행, 섹션 헤더가 더 중요함.
- Home에서도 감성 문장보다 **무엇을 어떻게 설계하는지**가 즉시 보이도록 해야 함.
- Works는 시각 이미지 썸네일보다 프로젝트의 정의, 포지션, 역할, 산출물, 결과를 표준화된 모듈로 제시하는 편이 적합함.
- 색상은 현재 요청에 맞게 거의 흑백으로 가져가되, 드림컴스 문서가 보여주는 **기준표적 사고방식**은 강하게 반영해야 함.

## 3. 엠플로컴퍼니 IR (`/home/ubuntu/upload/(주)엠플로컴퍼니_IR_20251030.pdf`)

### 관찰 범위
- 1~5페이지 확인

### 핵심 디자인 언어
이 문서는 가장 직접적으로 **투자 제안서 / IR 문서의 질서감**을 보여줍니다. 앞선 자료들보다 더 강한 검정 헤더, 더 명확한 좌우 분할, 더 실무적인 정보 정리 방식이 보입니다. 웹으로 확장할 때 중요한 기준점이 될 수 있습니다.

### 정보 구조
- 표지는 좌측의 핵심 메시지와 우측의 실제 사진으로 분할되어, 브랜드 비전과 현실 근거를 동시에 제시함.
- 이어지는 회사 개요 페이지는 검정 사이드 컬럼과 밝은 정보 영역을 명확히 분리해, 항목명과 값의 관계를 즉시 이해하게 함.
- 챕터 전환 슬라이드는 `01. ABOUT THE CEO` 같은 형식으로 매우 단순하게 구성되며, 웹에서도 큰 섹션 전환에 그대로 응용 가능함.

### 레이아웃 특징
- 검정 헤더 바 또는 검정 사이드 컬럼이 강한 구조적 프레임 역할을 함.
- 정보는 자유롭게 떠 있는 것이 아니라 **표, 행, 열, 메타 데이터 블록** 안에 정리됨.
- 이미지가 들어가더라도 주변에 충분한 정렬선과 여백이 있어 문서 전체가 차분하게 유지됨.

### 타이포그래피와 계층
- 굵은 산세리프 제목, 중간 굵기의 소제목, 가독성 높은 본문으로 위계가 명확함.
- 영문은 라벨·헤더·브랜드 표시에 제한적으로 쓰이고, 실제 정보 전달은 대부분 한글 중심으로 이뤄짐.
- 강세는 서체의 개성보다 **배치와 구조적 대비**에서 나옴.

### 정보 전달 방식
- 프로필, 경력, 수치, 그래프, 로고 배열이 모두 문서적 목적에 맞게 사용됨.
- 특히 경력 소개 페이지는 사진, 이름, 역할, 경력 목록, 대표 브랜드 로고를 논리적인 모듈로 묶어놓아, 감성 소개보다 **검증 가능한 이력 제시**에 가까움.
- 따라서 About 페이지 또한 감성적인 자기서사보다 **경험·역할·성과·관점**이 정리된 형식이 더 설득력 있을 수 있음.

### 웹 적용 시 해석 방향
- 사이트 전역에서 검정 헤더 바, 명확한 메타 라벨, 2열 정보 구조를 핵심 패턴으로 사용할 수 있음.
- Home 히어로도 감성 카피 중심보다, 좌우 분할 혹은 상단 문서 헤더 + 핵심 포지션 문장 + 근거 요약의 형태가 적합함.
- About과 Works는 프로필/프로젝트를 감상용 카드가 아니라 **문서형 브리프**로 보이게 해야 함.
- 정보가 많은 페이지일수록 표와 정의 리스트를 적극 도입하는 편이 이 PPT 언어와 잘 맞음.


## 4. Atlantis Strength Apparel Licensing Proposal (`/home/ubuntu/upload/AtlantisStrengthApparelLicensing_Proposal.pdf`)

### 관찰 범위
- 1~5페이지 확인

### 핵심 디자인 언어
이 문서는 앞서 본 제안서들 중에서도 가장 직접적으로 **라이선싱 제안서 / 사업 제휴 제안서**의 구조를 보여줍니다. 시각적으로는 단정하지만, 단순히 미니멀한 것이 아니라 **제목, 목차, 챕터 전환, 프로필 소개, 이력 검증**이 논리 흐름에 맞게 끊어지는 방식이 매우 명확합니다. 웹으로 옮길 때 중요한 것은 미감을 흉내 내는 것이 아니라, 이 문서가 가진 **정보 조직 방식**을 인터페이스 구조로 번역하는 일입니다.

### 정보 구조
- 표지에서 **인물 사진 + 제안 제목 + 제안 개요 + 연락처**를 한 화면 안에 정리합니다.
- 목차는 좌측의 검정 블록과 우측의 흰 콘텐츠 영역으로 분리되어, 처음부터 문서의 읽기 방식이 명확합니다.
- 챕터 전환 페이지는 `01. About EMFLOW COMPANY`처럼 **숫자 + 대제목 + 짧은 보조문장**만으로 구성되어 매우 절제되어 있습니다.
- 본문에서는 프로필, 경력, 이전 프로젝트, 로고 목록 등을 하나의 카드/패널 안에 넣어 **이력 검증형 정보 모듈**을 만듭니다.

### 레이아웃 특징
- 검정 상단 바가 페이지의 문맥을 고정합니다. 좌측에는 현재 장 제목과 소주제, 우측에는 회사 로고를 두어 문서가 항상 같은 시스템 안에 있다는 느낌을 줍니다.
- 본문은 흰 배경 위에 **테두리 있는 패널**을 두고, 그 안을 다시 좌우 2열로 나눕니다. 이는 웹에서 케이스 스터디나 About 페이지에 매우 유효한 구조입니다.
- 이미지, 이름, 직함, 경력 설명, 브랜드 로고가 한 패널 안에서 정렬되어 있어, 감성적 자기소개가 아니라 **신뢰 기반 소개 문서**처럼 읽힙니다.

### 타이포그래피와 계층
- 전반적으로 **산세리프 단일 계열**에 가까운 안정적 조합입니다.
- 제목은 크고 굵지만 과시적이지 않고, 본문은 작은 크기에서도 읽기 쉽게 정렬됩니다.
- 정보의 위계는 서체 장식이 아니라 **굵기 차이, 줄 간격, 블록 구조, 검정 헤더 바**로 형성됩니다.
- 영문 문서이지만 전달 방식은 매우 실무적이어서, 한글 사이트에도 거의 그대로 이식 가능한 구조입니다.

### 이미지와 텍스트의 균형
- 표지의 인물 이미지는 문서의 현실성을 높이되, 이후 페이지에서는 이미지가 감성 장치가 아니라 **증빙 자료나 인물 식별 장치**로 기능합니다.
- 로고 배열 역시 장식이 아니라 **실제 트랙 레코드의 증거**로 제시됩니다.
- 즉, 이미지 사용은 분위기 조성보다 신뢰 확보에 가깝습니다.

### 웹 적용 시 해석 방향
- Home 첫 화면에서 이미지가 들어가더라도 배경 장식이 아니라 **포지션을 증명하는 자료**처럼 보여야 합니다.
- About 페이지는 감성 자서전보다 **인물 소개 패널 + 경력 모듈 + 참여 브랜드/프로젝트 증거** 구조가 더 적합합니다.
- Works 역시 썸네일 카드보다 **프로젝트 브리프 패널** 형식이 어울립니다.
- 전역적으로 검정 헤더 바, 섹션 번호, 흰 본문 패널, 좌우 분할, 얇은 테두리 사용이 이 문서의 핵심 패턴입니다.
